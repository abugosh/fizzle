# ADR-016: Decompose events/game.cljs into focused sub-modules

## Status

Accepted

## Context

`events/game.cljs` is 903 lines braiding three independent concerns: the spell casting pipeline (~250 lines, ADR-015's `evaluate-pre-cast-step` multimethod), stack resolution (~93 lines), and priority/phase orchestration (~420 lines). It participates in 152 of 416 filtered commits over 6 months (37%), making it the highest-churn file in the codebase.

Three other game event concerns — phases, cleanup, and lands — have already been extracted to their own namespaces under `events/` and are re-exported through `game.cljs` for backward compatibility. The same pattern applies to the remaining three concerns.

Internal coupling analysis shows casting has zero dependencies on the other two concerns, resolution is called by priority only (one function), and priority orchestrates casting + resolution + phases + cleanup through clear interfaces. The decomposition follows existing seams.

## Decision

We will extract the three remaining concerns from `events/game.cljs` into focused sub-modules:

1. **`events/casting.cljs`** (~250 lines) — `::cast-spell`, `::select-casting-mode`, `::cancel-mode-selection`, `initiate-cast-with-mode`, `evaluate-pre-cast-step` multimethod and all 8 defmethods.
2. **`events/resolution.cljs`** (~100 lines) — `::resolve-top`, `::resolve-all`, `resolve-one-item`, and support functions.
3. **`events/priority_flow.cljs`** (~420 lines) — `::yield`, `::yield-all`, `::cast-and-yield`, `negotiate-priority`, `advance-with-stops`, bot-turn helpers.

`events/game.cljs` reduces to a backward-compatibility facade (~150 lines) that re-exports handlers from all sub-modules, following the precedent set by phases/cleanup/lands.

Multimethod continuation registrations (`:spell-mode`, `:cast-after-spell-mode`, `:resolve-one-and-stop`) move with their respective modules.

## Consequences

- Changes to casting logic no longer touch resolution or priority code
- Test files can target specific concerns rather than the monolithic game.cljs
- The re-export facade preserves all existing require paths — no downstream changes needed
- New game event concerns add a new sub-module instead of growing game.cljs
- The extraction order matters: casting first (zero deps), then resolution (one dep), then priority (calls both)
- More files under events/ — 6 sub-modules (phases, cleanup, lands, casting, resolution, priority_flow) plus game.cljs facade
