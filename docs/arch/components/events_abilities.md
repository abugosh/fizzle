# Ability Events

## Responsibility

re-frame event handlers for activated and triggered abilities on permanents. This component orchestrates the ability activation pipeline: validating that costs can be paid, collecting targeting selections when required, paying costs, and placing the ability on the stack. It also handles triggered ability queuing — abilities that fire automatically in response to game events (e.g., becomes-tapped triggers, land-entered triggers).

## Interface Contract

### IN

- re-frame events dispatched by Views / UI: `::activate-ability`, `::confirm-ability-target`
- Triggered ability events generated by the Engine during effect resolution or phase transitions
- Datascript db containing permanent state, cost availability, and valid target sets

### OUT

- Updated Datascript db with ability stack items and paid costs
- Downstream dispatches to Selection Events when ability targeting or modal cost choices are needed
- Datascript db updates after ability resolution (delegated to Engine)

## Local Changes

Adding a new activated ability cost type or a new trigger condition is local to this component as long as it uses existing Engine cost-validation and targeting primitives. Changes become non-local when a new ability type requires a new Engine function or a new Datascript schema attribute.
